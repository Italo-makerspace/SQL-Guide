Data Types: The Library's Drawers

Think of data types as different kinds of drawers for storing information. Each drawer has a specific format for what goes inside: one for numbers, another for text, another for dates, and so on. Choosing the right drawer is fundamental to keeping everything organized.

Numbers

  • INT: For integer numbers. Ex: age = 25
  • BIGINT: For very large integer numbers. Ex: social_security = 12345678901
  • DECIMAL(10,2): For numbers with a decimal point and exact precision, like currency. Ex: price = 49.90
  • FLOAT/DOUBLE: For numbers with a decimal point and approximate precision. Ex: height = 1.75
  • BOOLEAN: For true or false values (usually 1 or 0). Ex: is_active = 1

Date and Time

  • DATE: To store only the date. Ex: 2025-08-19
  • DATETIME: To store both date and time together. Ex: 2025-08-19 15:30:00
  • TIMESTAMP: Similar to DATETIME, but it is automatically updated on certain operations.
  • TIME: To store only the time. Ex: 15:30:00
  • YEAR: To store only the year. Ex: 2025

Text (Strings)

  • CHAR(n): For fixed-length text. Ex: product_code = 'AB123'
  • VARCHAR(n): For variable-length text (most common). Ex: first_name = 'Thierry'
  • TEXT: For very large blocks of text, like descriptions. Ex: description = "Product..."
  • ENUM: To choose a single option from a predefined list. Ex: status = 'yes'
  • SET: To choose multiple options from a predefined list. Ex: hobbies = 'a,c'

Binary Data

  • BLOB: To store large binary data, such as photos, PDFs, or music. Ex: profile_picture
